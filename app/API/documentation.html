GET /checkUser

Used to check the existence of some user.


Request user existence:

http://localhost:8000/checkUser/?email=asdas http://localhost:8000/checkUser/?telephone=11917247788

Parameters Email(string) Telephone(interger)

Obs.: One of these are required then when one are being used the other are ignored


Possible responses:

True = User founded

False = User not founded

NoData = Problem with parameters




GET /loginUser

Check if the password and email match and so providing a status response


Request login: http://localhost:8000/loginUser/?email=john@john.com&passwor=johna b


Parameters Email(String) * Password(String) *


Possible responses

UserNotFound = The email provided doesn’t to any user in database

True = The user is logged

False = Probably the email or password don’t match




POST /signUp

Use for create a account


Request to create account:

http://localhost:8000/signUp


Request Body


{

"email": "email@email.com", "password": "mypassword213", "name": "My name", "telephone": 0

}


Parameters Email(string) * Password(string) * Name(string) * Telephone(interger)


Possible responses:

TypeValidEmail = You probably insert a invalid email TypeNameError = You insert a name with less than 3 digits NumbersInNameError = A invalid number was sended AccountCreated = Your account was successfully created

AccountCreatedWithNumber = Your account was successfully created using a phone number

NoDataWasSend = You don’t send any data

ThisAccountAlreadyExists = The email that you send already is registered in database
